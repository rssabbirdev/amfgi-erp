# Client & supplier list API

Read-only JSON API for integrating **cheque clients** and **cheque suppliers** (including contacts) with another application.

This API does **not** use the normal browser login session. Access is controlled with a **Bearer token** issued from this app’s **Settings** screen.

---

## Security expectations

- Use **HTTPS** in production. Do not send the token over plain HTTP.
- Store the token only in a **secret store** or environment variable on the consuming side (never commit it to source control).
- The API is intended for **server-to-server** calls. This app does not send CORS headers for these endpoints; if you must call from a browser, use your own backend as a proxy or add CORS in a controlled way on your side.
- **Rotate** the token from Settings if it may have leaked. Rotation invalidates the previous token immediately.

---

## Obtaining a token

1. A user who can open **Settings** in this app (permission `admin.settings.manage`) goes to **Settings**.
2. In the section **API — client & supplier lists**, click **Generate API token** (first time) or **Rotate API token** (replace existing).
3. The raw token is shown **once** on the next page. Copy it immediately and give it to the integrating developer or store it in the other app’s secrets.

If no token has been generated yet, API requests receive `401` with a message that the API is not configured.

## Base URL

All paths are relative to the deployed app root, including any URL prefix.

| Deployment | Example base |
|------------|----------------|
| Site at domain root | `https://example.com` |
| App in a subpath (e.g. `/AccountsHelper`) | `https://example.com/AccountsHelper` |

Endpoints (always use **GET**):

- `{base}/api/v1/clients.php`
- `{base}/api/v1/suppliers.php`

Example:

```text
https://example.com/AccountsHelper/api/v1/clients.php
```

---

## Authentication

Send the token on every request using an **Authorization** header:

```http
Authorization: Bearer <your-token-here>
```

Rules:

- The scheme must be `Bearer` (case-insensitive), followed by a single space and the token.
- The token is a long hexadecimal string (64 characters) when generated by this app.
- Do not send cookies or session headers; they are not required and do not grant API access.

### Apache note

Some Apache setups do not pass `Authorization` to PHP by default. If requests always return `401` even with the correct token, the server may need configuration so `HTTP_AUTHORIZATION` is forwarded (common `SetEnvIf` / rewrite patterns). Nginx usually forwards the header without extra steps.

---

## Methods

Only **GET** is supported. Any other method returns **405** with JSON:

```json
{
  "error": "Method Not Allowed",
  "message": "Only GET is supported."
}
```

---

## Response format

- **Content-Type:** `application/json; charset=utf-8`
- **Encoding:** UTF-8; non-ASCII characters are not escaped as `\uXXXX` in JSON where possible.

### GET `clients.php` — success (200)

Top-level key **`clients`**: array of client objects, sorted by `name`.

```json
{
  "clients": [
    {
      "id": 1,
      "name": "Example Client Ltd",
      "email": "accounts@example.com",
      "trade_license_number": "…",
      "trade_license_authority": "…",
      "trade_license_expiry": "2026-12-31",
      "trn_number": "…",
      "trn_expiry": "2027-01-15",
      "created_at": "2026-01-15T10:00:00+04:00",
      "updated_at": "2026-02-01T12:00:00+04:00",
      "contacts": [
        {
          "id": 10,
          "contact_name": "Jane Doe",
          "email": "jane@example.com",
          "phone": "+971…",
          "sort_order": 0,
          "created_at": "2026-01-15T10:05:00+04:00"
        }
      ]
    }
  ]
}
```

- `email` on the client may be `null` if empty.
- `trade_license_expiry`, `trn_expiry`, and `updated_at` may be `null` if not set.
- Date/time fields are returned as stored in the database (typically ISO 8601 for timestamps; date fields may be `Y-m-d`).

**Omitted on purpose (not in the response):** internal document paths such as `trade_license_file` and `trn_file`. Those are filenames/paths on this server and must not be exposed to external systems.

### GET `suppliers.php` — success (200)

Same structure, with top-level key **`suppliers`** and the same field set per supplier (plus nested **`contacts`** from supplier contacts).

```json
{
  "suppliers": [
    {
      "id": 1,
      "name": "Example Supplier LLC",
      "email": null,
      "trade_license_number": "",
      "trade_license_authority": "",
      "trade_license_expiry": null,
      "trn_number": "",
      "trn_expiry": null,
      "created_at": "…",
      "updated_at": null,
      "contacts": []
    }
  ]
}
```

## Error responses

All errors are JSON (no HTML login pages for these endpoints).

| HTTP status | When |
|-------------|------|
| **401** | Missing/invalid `Authorization`, wrong token, or API token not configured on the server |
| **405** | Method other than GET |
| **500** | Server-side failure loading data (generic message; details are logged server-side) |

Example **401** (invalid token):

```json
{
  "error": "Unauthorized",
  "message": "Missing or invalid Bearer token."
}
```

Example **401** (token not configured):

```json
{
  "error": "Unauthorized",
  "message": "API access is not configured. Generate a token in Settings."
}
```

Example **500**:

```json
{
  "error": "Internal Server Error",
  "message": "Could not load clients."
}
```

## Support checklist for integrators

1. Confirm **base URL** (including path prefix) with the AccountsHelper host.
2. Obtain the **Bearer token** from an administrator (one-time display after generation).
3. Call only **GET**; handle **401** / **405** / **500** as above.
4. Prefer **HTTPS** and secure storage for the token.
5. If **401** persists with a known-good token, ask the host to verify Apache/header forwarding.

## Versioning

These endpoints live under **`/api/v1/`**. If breaking changes are introduced in the future, new paths (e.g. `v2`) may be added; this document describes **v1** only.

## Consuming from AMFGI (this ERP)

Server-side sync uses these **environment variables** (never commit real values; set in `.env` / deployment secrets):

| Variable | Example | Purpose |
|----------|---------|---------|
| `PARTY_LISTS_API_BASE_URL` | `https://example.com/AccountsHelper` | Deployed app root, **including** any path prefix (no trailing slash required). AMFGI appends `/api/v1/clients.php` or `suppliers.php` automatically. |
| `PARTY_LISTS_API_BEARER_TOKEN` | *(64-char hex from source app Settings)* | **Token only** — no `Bearer ` prefix (AMFGI adds it). If you pasted `Bearer …` or a full `Authorization:` line, it is normalized; double `Bearer` breaks validation on the party app. |
| `PARTY_LISTS_API_CLIENTS_URL` | *(optional)* | Full URL to the clients list endpoint if it is not `{base}/api/v1/clients.php`. |
| `PARTY_LISTS_API_SUPPLIERS_URL` | *(optional)* | Full URL to the suppliers list endpoint if it is not `{base}/api/v1/suppliers.php`. |

If either base URL or token is missing, **Sync from party API** on Customers / Suppliers will fail until they are set.

**404 when the requested URL already looks like** `…/AccountsHelper/api/v1/suppliers.php`: AMFGI built the path from the spec; the **remote server** returned “not found”. Check that the party app deployment includes `suppliers.php`, try `clients.php` with the same token, and fix hosting/rewrites on the party app side. You can set `PARTY_LISTS_API_SUPPLIERS_URL` (or clients) to a full URL that you have verified with curl.

**404 / HTML (wrong base):** Missing folder like `/AccountsHelper`, or domain-only base while the API lives under a subpath. If you pasted a full URL ending in `/api/v1/clients.php` into `PARTY_LISTS_API_BASE_URL`, AMFGI strips that suffix and rebuilds the path. Do **not** put `/api/v1` twice (e.g. base `…/AccountsHelper/api/v1` is normalized).

Sync is **read-only** into AMFGI’s database: creates/updates rows tagged as synced from the API. Records created in AMFGI stay **local** and can be deleted when unused; synced rows cannot be deleted from AMFGI (only edited or deactivated).
